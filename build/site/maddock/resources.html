Resources

A manifest is an ordered list of resources. Each resource has a type, a name, and a set of attributes. The engine processes them in the order they appear in the YAML.

name: webserver

resources:
  - package:
      nginx:
        state: present

  - file:
      /etc/nginx/nginx.conf:
        source: templates/nginx.conf.tmpl
        owner: root
        group: root
        mode: "0644"
        vars:
          worker_connections: 1024

  - service:
      nginx:
        state: running
        enabled: true

package

Manages a system package via apt on Debian/Ubuntu or dnf on Fedora. The package manager is auto-detected at startup.

Attributes:

state

present (install if missing) or absent (remove if present).

- package:
    htop:
      state: present

file

Manages a file’s content, owner, group, and mode. Writes are atomic (temp file + rename) so readers never see a partially-written file.

Attributes:

content

Inline content. Mutually exclusive with source.

source

Path to a Go text/template file. Mutually exclusive with content. Rendered at parse time using vars.

vars

Template variables. Used only with source.

owner

Unix username. Required.

group

Unix group name. Required.

mode

Octal permission string, e.g. "0644". Required.

- file:
    /etc/app.conf:
      content: |
        log_level=info
        port=8080
      owner: root
      group: root
      mode: "0644"

Templates receive vars as a flat map. Reference a variable with {{ .key }}:

- file:
    /etc/nginx/nginx.conf:
      source: templates/nginx.conf.tmpl
      owner: root
      group: root
      mode: "0644"
      vars:
        worker_connections: 1024

service

Manages a systemd unit.

Attributes:

state

running (start) or stopped (stop).

enabled

true (enable on boot) or false (disable).

- service:
    nginx:
      state: running
      enabled: true
