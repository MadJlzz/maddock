Maddock

Maddock is an infrastructure-as-code tool that converges Linux hosts to a desired state described in YAML manifests. Define packages, files, and services declaratively; Maddock handles the rest, idempotently.

How it works

  • The agent (maddock-agent) runs on each target host. It can apply manifests locally or serve a gRPC endpoint to receive catalogs pushed from a central server.

  • The server (maddock-server) reads a server config that maps hosts to manifests, dials each agent in parallel, and collects reports.

Quick start

Install the agent:

curl -fsSL https://raw.githubusercontent.com/MadJlzz/maddock/main/install.sh | sh

Write a manifest:

name: webserver

resources:
  - package:
      nginx:
        state: present
  - service:
      nginx:
        state: running
        enabled: true

Apply it:

sudo maddock-agent apply webserver.yaml

Run twice — the second run reports every resource as OK because the host is already in the desired state. That’s convergence.
